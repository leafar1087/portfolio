# Automation with Python and Cloudflare

Today I implemented a **Serverless** architecture for my professional portfolio. The goal was to avoid heavy content managers (like WordPress) and maintain total control over security and performance.

## The Architecture

I utilized the JAMstack:

1.  **Javascript**: For dynamic client-side rendering.
2.  **APIs (JSON)**: Automatically generated by Python.
3.  **Markup**: Content written purely in Markdown.

## The Deployment Pipeline

Every time I `git push`, Cloudflare Pages detects the changes and executes a custom Python script:

```python

def parse_frontmatter(content):
    """
    Extracts metadata from the YAML frontmatter block.
    """
    # Regex para capturar el contenido entre los separadores '---'
    match = re.search(r'^---\s*\n(.*?)\n---\s*', content, re.DOTALL)

    if not match:
        return None

    frontmatter_raw = match.group(1)
    metadata = {}

    # Parseo línea por línea
    for line in frontmatter_raw.split('\n'):
        # ... lógica de limpieza y extracción ...
        parts = line.split(':', 1)
        if len(parts) < 2: continue

        key = parts[0].strip()
        value = parts[1].strip()
        metadata[key] = value

    return metadata
```

This approach allows me to maintain bilingual logic (Spanish/English) and date sorting directly from the "Build Time", without impacting the user's browser performance.

## Security Benefits

By not having a live SQL database or running PHP server:

1. I eliminate SQL Injection vulnerabilities.
2. I mitigate brute force attacks on admin panels (no panel exists).
3. I drastically reduce the attack surface.

# Automatización con Python y Cloudflare

Hoy he implementado una arquitectura **Serverless** para mi portafolio profesional. El objetivo era evitar gestores de contenido pesados (como WordPress) y mantener un control total sobre la seguridad y el rendimiento.

1.  **Javascript**: Para la renderización dinámica en el cliente.

3.  **Markup**: Contenido escrito puramente en Markdown.

## El Pipeline de Despliegue

Cada vez que hago un `git push`, Cloudflare Pages detecta los cambios y ejecuta un script de Python personalizado:

Este enfoque me permite mantener la lógica bilingüe (Español/Inglés) y el ordenamiento por fechas directamente desde el "Build Time", sin impactar el rendimiento del navegador del usuario.
